# 青程科创官网项目介绍

## 项目定位

青程科创官网是“青程科创”团队的宣传与功能型网站，用于展示团队理念、科创营课程、学员成果、工具入口、动态内容和报名咨询入口。

网站不是单纯的公司介绍页，而是面向学校、家长、研学机构和团队成员的综合入口：

- 对外：展示青程科创的教育理念、项目体系、真实成果和合作入口。
- 对内：沉淀课程 SOP、作品展墙、积分统计、课程迭代资料和运营工具入口。
- 对招生：提供开放营期、课程亮点、家长/学校常见问题和咨询表单。

## 核心主张

青程科创希望让青少年在真实项目中理解技术、表达想法、完成作品。

网站围绕“从真实问题走向真实创造”展开，强调：

- 真实任务：从造船、发射水火箭、调试机器人、制作网页等真实挑战出发。
- 工程实践：让学生经历设计、制作、测试、失败、迭代和展示的完整过程。
- AI 协同：把 AI 作为工程伙伴，用于资料整理、方案分析、代码辅助和表达优化。
- 成果证据：用作品、数据、工程日志、展示视频和成长档案证明学习效果。

## 网站主要板块

### 1. 首页首屏

展示青程科创品牌名称、核心定位、项目速览和行动入口。

首屏包含：

- 品牌标题：青程科创
- 核心文案：从真实问题走向真实创造
- 数据速览：核心课程线、最长营期学时、SOP 交付能力
- 行动按钮：提交营期需求、查看开放营期

### 2. 教育理念

介绍青程科创为什么做项目式科创营。

该板块强调官网参考了“教育品牌 + 项目招生系统 + 学员作品展示 + 工具平台 + 内容运营阵地”的闭环，不只做展示页，而是逐步发展为可运营的网站系统。

### 3. 青程 CRAFT 模型

网站将课程方法论整理为 CRAFT 模型：

- Challenge：从真实挑战开始
- Research：观察、实验、资料检索和 AI 协作
- Assembly：完成结构、硬件、网页或数字作品
- Feedback：通过测试和数据进行迭代
- Tell：公开展示、答辩和作品传播

### 4. 营期项目

当前官网展示四条课程线：

- AI 仿生机器人科创营
- 二级水火箭半日营
- 《让知识浮起来》七天工程创造营
- 个人网页创作营

每个课程卡片包含适合对象、课程产出和项目亮点。

### 5. 开放营期

展示“夏日 STEM + 运动研学营”主题，嵌入水火箭招生海报，用于后续招生转化。

该板块适合放：

- 当前开放报名的营期
- 招生对象
- 营期主题
- 安全保障
- 合作咨询入口

### 6. 体验升级展示

展示水火箭课程中的体验增强组件，包括：

- 彩色尾迹 GIF
- 风螺声音展示 MP4
- 发射仪式感
- 数据记录
- 颁奖与成果传播

### 7. 七天工程创造营

根据《让知识浮起来_七日科创营课程方案.docx》整理。

课程主题为“从 0 到 1 造一艘船”，面向高中生，核心流程包括：

- Day 1：工程师是怎样思考的？
- Day 2：为什么船能浮起来？
- Day 3：为什么船会翻？
- Day 4：AI 工程师上线
- Day 5：从图纸到现实
- Day 6：测试与迭代
- Day 7：工程创造者大会

课程强调完整工程闭环、AI 协同、CAD/3D 打印、测试迭代和公开答辩。

### 8. 成果展示

展示学生网页作品、团队创作、积分看板等内容，并链接到已有成果展示页。

### 9. 青程工具台

当前接入本地已有工具和资料：

- 积分统计
- 学生作品展墙
- 助教执行手册
- 课程迭代库

后续可以独立扩展为类似工具站的系统。

### 10. 动态、反馈与档案

用于承载营期记录、课程研发动态、家长反馈和证书/作品档案查询。

当前证书查询为前端演示，后续可接入数据库或飞书表格。

### 11. 报名咨询

网站内置咨询表单，可记录：

- 联系人
- 联系方式
- 预计人数
- 感兴趣项目
- 需求说明

当前表单数据保存到浏览器本地，可导出 JSON。后续可接入飞书多维表格、数据库或表单服务。

## 技术结构

本项目是纯静态网页，不依赖构建工具。

文件结构：

```text
青程科创官网/
├── index.html          # 网站页面结构
├── styles.css          # 样式与响应式布局
├── script.js           # 交互逻辑与动画
├── 项目介绍.md          # 项目介绍文档
├── assets/             # 图片、GIF、视频、DOCX 等素材
```

## 部署方式

由于项目是静态网页，可以直接使用 GitHub Pages 部署。

推荐方式：

- 仓库根目录放置 `index.html`
- GitHub Pages 来源选择 `main` 分支 `/root`
- 发布后访问地址通常为：

```text
https://你的用户名.github.io/仓库名/
```

如果仓库名是 `你的用户名.github.io`，则可以作为个人主页访问：

```text
https://你的用户名.github.io/
```
